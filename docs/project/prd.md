# BidMaster (비드마스터) — PRD

## 1. 프로젝트 개요

- **목적**: AI 기반 공공 입찰 제안서 자동화 SaaS — 대한민국 중소기업이 AI를 활용해 공공 입찰 제안서를 스스로 작성하고, 낙찰 가능성을 데이터로 판단할 수 있는 입찰 자동화 플랫폼
- **대상 사용자**: 중소 IT 기업 대표/영업팀장, 입찰 담당자, 제안서 대행 프리랜서
- **핵심 가치**: 제안서 작성 비용 90% 절감, 낙찰 의사결정 데이터화, 제안서 품질 표준화

### 문제 정의

| 문제 | 규모 | 임팩트 |
|------|------|--------|
| 공공조달 시장 연간 200조원 규모 | 나라장터 등록 기업 50만개 이상 | 거대한 타깃 시장 |
| 제안서 대행업체 비용 건당 300만~2,000만원 | 중소기업 연간 수천만원 지출 | 핵심 비용 절감 포인트 |
| 낙찰률 10~30% — 비용 대비 불확실한 ROI | 낙찰 실패 시 비용 전액 손실 | 리스크 집중 |
| 공고 분석에 담당자 1~3일 소요 | 놓치는 적합 공고 다수 발생 | 기회 손실 |

### 포지셔닝

| 구분 | 기존 대행업체 | **비드마스터** |
|------|------------|-------------|
| 제안서 비용 | 300만~2,000만원/건 | **30~100만원/건** |
| 공고 분석 속도 | 2~3일 | **실시간 자동 매칭** |
| 낙찰 가능성 분석 | 경험 기반 주관적 | **데이터 기반 스코어** |
| 서비스 속도 | 1~2주 | **24시간 이내** |

## 3. 비기능 요구사항

### 성능
- RFP 텍스트 추출 성공률 95% 이상
- 매칭 점수 산출 시간 30초 이내
- 스코어링 계산 시간 60초 이내
- 10페이지 기준 초안 생성 시간 3분 이내
- 편집 지연(Latency) 100ms 이하

### 보안
- JWT 기반 인증 (액세스 토큰 1시간, 리프레시 토큰 30일)
- 비밀번호 bcrypt 해싱
- HTTPS 통신 필수
- 개인정보 암호화 저장

### 확장성
- 마이크로서비스 아키텍처 (크롤러, AI 서비스 분리)
- Redis 캐싱으로 빈번한 조회 최적화
- pgvector를 활용한 임베딩 검색 확장

## 4. 기술 스택

- **Backend**: FastAPI (Python 3.12)
- **Frontend**: Next.js 14 (App Router)
- **Database**: PostgreSQL + pgvector
- **Cache/Queue**: Redis
- **AI**: Claude API (제안서 생성, 스코어링)
- **Storage**: AWS S3
- **Payment**: 토스페이먼츠
- **Deploy**: Docker + Kubernetes (Railway/Vercel)

## 5. 범위 외 (Out of Scope)

- 비IT 분야 공고 (MVP 이후 확장)
- 실제 입찰 제출 대행
- 법률 자문
- 낙찰 후 계약 관리
- 공사/물품 입찰 (초기)
