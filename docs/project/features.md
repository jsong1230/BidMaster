## 인수조건

### F-01: 공고 자동 수집 및 매칭

**개요**
- **우선순위:** P0
- **카테고리:** 핵심
- **의존성:** F-08 (회사 프로필이 있어야 매칭 가능)

**인수조건**
- [ ] Given 나라장터에 신규 공고가 등록됨 When 크롤링 주기(06:00, 12:00, 18:00) 도달 Then 공고가 DB에 저장됨
- [ ] Given 공고에 PDF/HWP 첨부파일이 있음 When 파싱 실행 Then 텍스트가 추출되어 stored
- [ ] Given 회사 프로필이 등록됨 When 신규 공고 매칭 분석 Then 적합도 점수(0~100)가 산출됨
- [ ] Given 매칭 점수 70점 이상인 공고가 있음 When 분석 완료 Then 알림이 발송됨
- [ ] Given 크롤링 타임아웃 발생 When 재시도 로직 실행 Then 지수 백오프로 3회 재시도 후 실패 시 관리자 알림

**기술 고려사항**
- 나라장터 API/스크래핑 연동
- PDF/HWP 파싱 (pdfplumber, hwp 라이브러리)
- pgvector를 활용한 임베딩 기반 매칭
- 크론잡 또는 Celery Beat 스케줄링

### F-02: 낙찰 가능성 스코어링

**개요**
- **우선순위:** P0
- **카테고리:** 핵심
- **의존성:** F-01, F-08

**인수조건**
- [ ] Given 공고와 회사 프로필이 준비됨 When 스코어링 요청 Then 60초 이내에 낙찰 가능성 점수(0~100) 반환
- [ ] Given 스코어링 완료 When 결과 조회 Then 항목별 점수 분해 (경쟁강도, 역량, 적합도, 시장환경) 표시
- [ ] Given 스코어 70점 이상 When 추천 표시 Then "참여 추천" AI 의견 제공
- [ ] Given 스코어 50점 미만 When 추천 표시 Then "비추천" 및 사유 표시
- [ ] Given 과거 낙찰 데이터 존재 When 분석 Then 예상 경쟁사 리스트 제공

**기술 고려사항**
- 스코어링 모델 설계 (가중치 기반)
- 나라장터 과거 낙찰 데이터 수집/분석
- Claude API 활용한 AI 의견 생성

### F-03: 제안서 AI 초안 생성

**개요**
- **우선순위:** P0
- **카테고리:** 핵심
- **의존성:** F-01, F-08

**인수조건**
- [ ] Given 공고 선택 및 회사 프로필 존재 When "제안서 생성" 클릭 Then RFP 분석 후 섹션별 초안 생성 시작
- [ ] Given 제안서 생성 중 When SSE 스트림 연결 Then 섹션별 생성 진행 상황 실시간 표시
- [ ] Given 10페이지 기준 제안서 When 생성 완료 Then 3분 이내 완료
- [ ] Given 제안서 생성 완료 When 평가 항목 체크 Then 90% 이상 항목 반영 확인
- [ ] Given 제안서 다운로드 요청 When Word/PDF 선택 Then 10초 이내 다운로드 링크 제공
- [ ] Given RFP 파싱 실패 When 수동 입력 Then 텍스트 직접 입력 폼 제공

**기술 고려사항**
- Claude API 연동 (프롬프트 엔지니어링)
- SSE (Server-Sent Events) 스트리밍
- python-docx, reportlab으로 Word/PDF 생성
- 템플릿 엔진 (Jinja2)

### F-04: 낙찰가 예측 및 투찰 전략

**개요**
- **우선순위:** P0
- **카테고리:** 핵심
- **의존성:** F-01

**인수조건**
- [ ] Given 공고 선택 When 낙찰가 분석 요청 Then 유사 공고 낙찰가율 분포 시각화
- [ ] Given 분석 완료 When 결과 조회 Then 추천 투찰가 범위(안전구간) 제공
- [ ] Given 낙찰 방식이 적격심사 When 전략 제안 Then 예산 88~95% 구간 추천
- [ ] Given 최저가 낙찰 방식 When 전략 제안 Then 경쟁사 최저 투찰가 예측 포함
- [ ] Given 투찰가 시뮬레이션 When 금액 입력 Then 해당 금액 낙찰 확률 표시

**기술 고려사항**
- 통계 분석 (평균, 표준편차, 분위수)
- 과거 낙찰 데이터 DB 구축

### F-05: 제안서 편집기

**개요**
- **우선순위:** P0
- **카테고리:** 핵심
- **의존성:** F-03

**인수조건**
- [ ] Given 제안서 열람 When 편집기 로드 Then 리치텍스트 편집 가능 (서식, 표, 이미지)
- [ ] Given 편집 중 When 30초 경과 Then 자동 저장 실행
- [ ] Given 섹션 선택 When AI 재생성 클릭 Then 커스텀 지시사항 입력 후 재생성
- [ ] Given 평가 기준 패널 When 섹션별 체크 Then 달성률 실시간 업데이트

**기술 고려사항**

**개요**
- **우선순위:** P0

- **의존성:** F-01, F-07

**인수조건**

### F-07: 사용자 인증 및 계정 관리

**개요**
- **우선순위:** P0
- **카테고리:** 인프라
- **의존성:** 없음

**인수조건**
- [ ] Given 이메일/비밀번호 입력 When 회원가입 요청 Then 계정 생성 및 JWT 발급
- [ ] Given 로그인 요청 When 인증 성공 Then 액세스 토큰(1시간) + 리프레시 토큰(30일) 발급
- [ ] Given 액세스 토큰 만료 When API 요청 Then 401 응답
- [ ] Given 리프레시 토큰 When 갱신 요청 Then 새 액세스 토큰 발급
- [ ] Given 카카오 로그인 요청 When OAuth 인증 완료 Then 계정 연동 및 JWT 발급
- [ ] Given 로그아웃 요청 When 처리 Then 리프레시 토큰 무효화

**기술 고려사항**
- JWT 발급/검증 (python-jose)
- 비밀번호 해싱 (bcrypt)
- 카카오 OAuth 2.0 연동
- 토큰 블랙리스트 (Redis)

### F-08: 회사 프로필 관리

**개요**
- **우선순위:** P0
- **카테고리:** 인프라
- **의존성:** F-07

**인수조건**

- [ ] Given 회사 등록 완료 When 수행 실적 등록 Then 과거 사업 정보(발주처, 금액, 기간) 저장

**기술 고려사항**
